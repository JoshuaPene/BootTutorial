Introduction

Bootstrap is “the world’s most popular framework for building responsive, mobile-first sites.” Bootstrap optimizes code for mobile devices first and scales up web components through CSS. By offering pre-set classes, you can cut out a ton of CSS styling! Bootstrap has made a ton of preset-classes and their website is full of information on how to implement their code. While you can definitely check it out, I am going to teach you how to create a simple website that implements Bootstrap’s column layouts with some fun css layouts (don’t worry, it’s super easy to learn how to use)!

Considerations

Bootstrap requires the use of the HTML5 doctype. Without it, you’ll see some funky incomplete styling, but including it shouldn’t cause any considerable hiccups.



<!doctype html>

<html lang="en">

...

</html>


To ensure proper rendering and zooming for all devices, add the responsive viewport meta tag to your head.



<meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">

Bootstrap Explained

Now that you have bootstrap, I will talk briefly about what it is and how you can use it! Basically, Bootstrap offers pre-made classes that you have just linked to your code. From this, all you need to do is figure out what classes you want to add (classes can be stacked easily)! It is very simple and the possibilities are many, but it can seem overwhelming to start. I will be going over how to recreate my website with the classes I decided to implement in my code.

Navigation

First off, I am going to go over some of Bootstrap’s Navigation classes. (First you will need to create a header and nav class.) Bootstrap classes all have specific names that will perform their specific styles. In the Nav class that is set up, I have implemented many specific classes which I will show down below. First off, I will declare that the nav class is a “navbar”. This will let bootstrap apply some base code to the navbar that I want set up. Next I will show many different classes and what each one does!



Adding any of these styles to the class will change the styling of the navigation bar. Feel free to try a bunch of these and figure out what each one does and how each one can work with each other!

  • fixed-top:

    This will do exactly what it says! It will make the Nav-bar stick to the top of the screen (the navbar on this website is an example of this).

  • navbar-expand-sm (or xs, md, lg, xl):

    This will make sure that the navigation bar will expand due to the size of the screen that is being used. These are called responsive breakpoints. They will create breakpoints for the layouts and interfaces depending on the viewports scale. If you scale this website, you will see how the navbar reacts to the size restraints.

  • navbar-dark (navbar-light, etc.):

    This will change the color and style that you are using for the bar.

  • bg-dark (bg-light):

    This will change the background color used for the nav bar.

  • justify-content-between:

    You can also add stylings like "justify content" among other things!

  • collapse navbar-collapse:

    A component which wraps the remain components of the navbar such as links to other pages on the websites.

Href Classes

Now, one of the most exciting parts of Bootstrap is its ability to implement grid structures for you! Bootstrap uses containers, rows, and columns to layout and align content by using flexbox. In order to apply these grids, you have to put all of your elements into a div class named “container”. This is the first step for getting grids into your code. Bootstrap uses a 12-column, flexbox grid system to help the user implement tons of their classes.

  • container

    Will create a simple grid structure for you to begin working with through a column.

  • container-fluid

    WIll have a container span the width of the viewport.

  • row

    Rows are wrappers for columns.

  • column

    column classes indicate the number of columns you would like to have in a row out of a possible 12 per row. You will put column classes inside of row classes to make however many columns span the row. A column class will be typed as “col”.

    Bootstrap will automatically default the column settings to be equal-width. That will make your content in a row be spaced evenly.

    In order to define how many columns you want to stretch out over a certain class, you will add a number to the end of a col. (Ex. “col-4”).

    You have to make sure the numbers that you use throughout a row’s columns are equal to 12!

    You can also add a “-auto” to the end of a col to make the spacing even out for you so you don’t have to worry about how it looks at certain viewports as much.

Cards

To present my grid content, I chose to apply a simple “card-deck” to my classes to display my information in a really neat way. Cards are flexible and extensible content containers. They are built with “as little markup and styles as possible” so you have a lot of room to implement your own customization. I will show some types of content types that Bootstrap offers:

  • card-body

    The building block of a card. Basically just a body tag, but inside a card.

  • card-title

    A title inside your card. Likely applied to some kind of h4, h5, etc.

  • card-subtitle

    Subtitle under the main title.

  • card-text

    Applied to a p-class for your text to be affected.

  • card-link

    Used to be applied to a href.

  • card-img-top

    This places an image to the top of the card. With .card-text, text can be added to the card. Text within .card-text can also be styled with the standard HTML tags.
